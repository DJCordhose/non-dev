Was ist Programmieren?

  • Programmierer schreiben bzw. verändern Text-Dateien
  • Diese Text-Dateien enthalten die Beschreibung von Programmteilen
  • Solche Beschreibungen sind in einer bestimmten Programmiersprache geschrieben
  • Ein komplexes Programm besteht aus vielen solchen Dateien
  • Oft kommt mehr als eine Programmiersprache zum Einsatz
  • Programmierer benutzen häufig sehr komplexe, für ihre Bedürfnisse angepasste Text-Editor-Programme
  • Es ginge aber auch mit Notepad o.ä.

HTML-Grundlagen

  • Alle Seiten, die ihr euch im Internet anguckt sind in HTML geschrieben
  • HTML besteht aus
    • Text, z.B. "Hallo, Welt"
    • Struktur, z.B. <h1>...</h1> und
    • Layout, z.B. <h1 style="font-size: 100px">...</h1> und
    • evtl. Funktionalität, z.B. alert("Hallo, Welt");

HTML-Struktur: Tags

  • Struktur wird über sogenannte Tags angegeben
  • Tags beginnen mit einem Kleiner-Zeichen < und enden mit einem Größer-Zeichen >
  • Dazwischen steht immer ein Name, der den Zweck des Tags beschreibt, z.B.
    • h1: Heading 1, die größte Überschrift
    • h2: Heading 2, die nächstkleinere Überschrift
    • p: Paragraph, ein Absatz

HTML-Struktur: Beginn und Ende

  • Meist wird etwas Text durch zwei Tags umschlossen
  • Bei <h1>...</h1> gibt das erste Tag den Start der Überschrift, das zweite das Ende der Überschrift an
  • Ein Ende-Tag hat zusätzlich einen Slash / vor seinem Namen
  • Die Namen von Start- und Ende-Tag müssen übereinstimmen

HTML-Struktur: Attribute

  • Start-Tags können zusätzlich zu ihrem Namen noch weitere Informationen enthalten
  • z.B. gibt <h1 style="font-size: 100px">...</h1> die Größe der Schrift an
  • Diese Informationen heißen Attribute
  • Attribute haben ebenso einen Namen, der ihre Funktion beschreibt
  • Zusätzlich können Attribute auch einen Wert haben, der in Anführungszeichen steht
  • Attribute gelten bis zum Ende-Tag

JavaScript

  • Wir betrachten die Programmiersprache JavaScript
  • Diese Programmiersprache läuft auf jedem Computer, der einen Webbrowser enthält
  • Ihr könnt eigene HTML-Seiten mit JavaScript mit Leben füllen
  • JavaScript ist weltweit eine der am weitest verbreiteten Programmiersprachen

Anweisungen

  • Programme in JavaScript bestehen häufig aus Anweisungen
  • Mit Anweisungen teilt ihr dem Computer mit, was er tun soll
  • Ihr müsst dem Computer Anweisungen in einer bestimmten Form geben, damit er sie versteht
  • Der Computer ist sehr kleinlich

Beispiele

  • Der Computer soll eine Zahl ausgeben 
    console.log(10);
  • Der Computer soll 10 plus 10 ausrechnen 
    console.log(10 + 10);
  • Am Ende jeder Anweisung steht ein Semikolon
  • Jede Anweisung sollte in einer eigenen Zeile stehen
  • Ihr könnt auch mit Texten arbeiten, diese müssen dann in Anführungszeichen stehen 
    console.log("Olli hat Kaffeedurst");

Übung

  • Öffnet in jsbin.com die JavaScript- und Consolen-Reiter
  • Gebt das Ergebnis einer komplexe Berechnung mit console.log aus
  • z.B. console.log(10 * (10 + 29));
  • Lasst euren Namen ausgeben
  • z.B. console.log("Olli");
  • Lasst die Länge eures Namens ausgeben (console.log("Olli".length);)
  • z.B. console.log("Olli".length);
  • Versucht, Blödsinn auszuführen und guckt was passiert

Funktionsaufrufe

  • Damit richtig etwas passiert, müsst ihr Funktionen aufrufen
  • Eine Funktion ist wie ferngesteuertes Miniprogramm
  • Mehrere Anweisungen werden darin zusammengefasst
  • Funktionen haben einen Namen und erwarten eine Reihe von Parametern
  • Die Parameter geben der Funktion Informationen darüber, was im einzelnen passieren soll
  • Ihr ruft eine Funktion auf, indem ihr den Namen der Funktion angebt und in runden Klammern danach die Parameter

Beispiele

  • Der Computer soll 10 plus 10 nicht nur ausrechnen, sondern auch ausgeben
  • Wir rufen die Funktion alert auf und übergeben ihr als Parameter, was sie ausgeben soll 
    alert(10 + 10);
  • Diese Ausgaben erscheinen dann in einem Fenster
  • Funktionen können auf Werte zurück gegen, z.B. 
    console.log(encodeURIComponent("Hallo, Welt"));

Variablen

  • Wie in der Mathematik gibt es in der Programmierung Variablen
  • Variablen können unterschiedliche Werte annehmen und sind veränderbar
  • Ihr greift auf Variablen über ihre Namen zu
  • Variablen können z.B. Texte, Zahlen oder Objekte enthalten

Beispiele

  • Um eine neue Variable einzuführen, schreibt ihr das Wort var vor den Namen der Variablen
  • Die Variable laenge hat den Wert 10 
    var laenge = 10;
  • Ihr könnt den Wert einer Variablen auch ändern 
    laenge = 20;
  • Ihr könnt auf den Wert einer Variablen jederzeit wieder zugreifen, z.B. um ihn einer Funktion als Parameter zu übergeben 
    alert(laenge);

  • Gebt den Wert der Variablen auf der Console aus
  • Ändert den Wert der Variablen mit dem Rückgabewert einer Funktion
  • Gebt den Wert der Variablen erneut auf der Console aus

Wie organisiert man Daten?

  • Bezeichner anlegen, z.B.
    • Namen für Dateien
    • Kundennummern
  • Bei großen Datenmenge Hierarchien Anlegen
    • Ordner und Unterordner
    • Nummerierungen

Passende Datenstrukturen als Programmierkonzept

  • Objekte (vgl. Ordner)
  • Variablen mit Namen (vgl. Namen für Dateien)
  • Arrays (vgl. nummerierte Listen)

Datenstruktur: Objekt

  • Objekte haben wie Ordner Namen
  • Ein Objekt kann z.B. der Bildschirm sein
  • Objekte können wie Ordner verschachtelt sein
  • Z.B. ein Formular oder eine Spalte innerhalb des Bildschirms
  • Pfade können auf bestimmte Objekte verweisen
  • Anstatt \ bzw. / nehmt ihr bei Objekten den Punkt: .

Objekte in HTML

  • In HTML steht document für den aktuellen Bildschirm
  • Dieser Bildschirm kann u.a. Formulare für Eingaben enthalten (<form>)
  • Formulare können wiederum Eingabefelder enthalten (<input>)

Formular-Beispiel

HTML

<form name="ausgabe">
        <input name="laenge">
</form>

Setzen des Eingabefelds mit JavaScript über dessen Pfad

document.forms.ausgabe.laenge.value = 10;

So bettet ihr das JavaScript direkt in HTML ein

action="javascript:document.forms.ausgabe.laenge.value = 10"

Eigene Funktionen schreiben

  • Funktionen fassen mehrere Zeilen Programmiercode zusammen
  • Funktionen werden über ihren Namen definiert und können eine Reihe von Parametern erwarten
  • Die Parameter werden über eine Liste von Namen definiert
  • Innerhalb der Funktion verhalten sich die übergebenen Parameter wie Variablen
  • Eine Funktion kann beliebige Werte als Rückgabe liefern

Beispiel

  • Ihr beginnt eine Funktionsdefinition mit dem englischen Wort function
  • Dann kommen Name, Klammern und eine Liste von Namen, die die Parameter angeben
  • Die Programmierzeilen, die die Funktion zusammenfasst, stehen in geschweiften Klammern { ... } 
    function hallosagen(name) {
    alert("Hallo " + name);
}
  • nun könnt ihr die Funktion aufrufen und es erscheint ein Fenster mit dem Text "Hallo, Olli" 
    hallosagen("Olli");

Beispiel mit Rückgabe

  • Eine Funktion kann ein Ergebnis liefern, indem sie den Wert einer Variablen mit return zurück gibt, z.B. 
  • function hallotext(name) {
    var text = "Hallo " + name;
    return text;
}
  • Rückgabewerte können in Variablen abgelegt werden 
  • var text = hallotext("Olli");
console.log(text);

  • Lass die Funktion die beiden Parameter addieren und speichere das Ergebnis in einer Variablen
  • Nutzt dieses Ergebnis als Rückgabe-Wert der Funktion
  • Ruft die Funktion auf und gebt das Ergebnis auf der Console aus

Fallunterscheidungen

  • Manchmal will man anhand einer bestimmten Bedingung mal das eine Stück Code, mal ein anderes ausführen
  • Dazu gibt es in JavaScript eine Fallunterscheidung
  • Bei dieser kann man ein Stück Code angeben, wenn eine Bedingung erfüllt ist
  • ... und einen anderen, wenn die Bedingung nicht erfüllt ist
  • Fallunterscheidungen könnt ihr verschachteln

Beispiele #1

  • Ihr beginnt eine Funktionsdefinition mit dem englischen Wort if
  • Dann kommt in Klammern die Bedingung
  • Die Programmierzeilen, die dann ausgeführt werden sollen, stehen in geschweiften Klammern { ... } 
    if (name == "Olli") {
    alert("Hallo, Meister!");
}
  • Mit ==, <, >, <=, >= vergleicht ihr Werte

Beispiele #2

Mit else könnt ihr angeben, was passieren soll, wenn die Bedingung nicht erfüllt ist

if (name == "Olli") {
    alert("Hallo, Meister!");
} else {
    alert("Hallo!");
}

Pair-Programming

  • Beim Pair-Programming schreibt ihr mit jemand anderem zusammen
  • Ein Programmierer tippt, der andere berät und behält den Überblick
  • Die Rollen werden regelmäßig getauscht
  • Sinn ist hauptsächlich
    • eine Überprüfung des Programms
    • Wissenstransfer

Übung: Erzeuge deine eigene Magische Zahl

Wir spielen ein Ratespiel

  • Versuche herauszufinden, wie sich die Magische Zahl errechnet
  • Suche die einen Partner, mit dem du zusammen im Pair-Programming-Stil ein eigenes Rätseln erstellen willst
  • Beginne mit dem Ratespiel auf jsbin.com
  • Verändere die Berechnung der geheimen Zahl, so dass das erraten möglichst schwierig ist
  • Stelle allerdings sicher, dass du für jede Eingabe das richtige Ergebnis im Kopf errechnen kannst
  • Teile Links auf das fertige Programm und lass andere deine Berechnung erraten

Hilfestellungen

  • Lass dich nicht von dem komplexen HTML einschüchtern, dies musst du nicht versehen
  • Verändere nur die Funktion werteFormularAus, die bereits vorliegt
  • Die Funktion parseInt macht aus einem Text eine Zahl
  • Nutze eine ggf. Fallunterscheidung und die Berechnung schwieriger zu machen

Eigene Objekte erzeugen

  • Ihr könnt auch eigene Objekte erzeugen, um Daten zu strukturieren
  • Objekte können Texte, Zahlen und sogar Funktionen enthalten
  • Alle Inhalten von Objekten können jederzeit geändert werden

Beispiel: Ein neues Objekt erzeugen

var box = {
    x: 0,
    y: 0,
    w: 10,
    h: 10,
    color: 'black',
    speed: 5
};
box.x = 100;
